# Milestone v7.0: Plan 1 Foundation

**Status:** SHIPPED 2026-06-30
**Phases:** 7-9
**Total Plans:** 8

## Overview

This milestone completed the Plan 1 foundation for VeloxMesh's architecture refactor: SQLite as durable relational state, Redis Stack for hot-path coordination and rate/cache state, Qdrant as the primary vector path, and Redis VSS as a fallback path when Qdrant is degraded.

Phase 10-13 remain future roadmap items and are not included in this milestone close.

## Phases

### Phase 7: Adapter Interfaces & SQLite Foundation

**Goal:** Make the v2.1 Plan 1 runtime explicit: SQLite authoritative relational state, Redis Stack hot state, Qdrant primary vector store, and narrow adapters for future extensions.
**Depends on:** Phase 6
**Plans:** 1

Delivered:
- SQLite-first durable startup and validation.
- Narrow storage adapter contracts.
- Fallback log schema.
- Qdrant vector and semantic-cache foundation.
- LanceDB retained only for future edge builds.

### Phase 8: Semantic Pipeline

**Goal:** Implement configurable input/output processing with handler registry, rule toggles, and hot-reloadable configuration.
**Depends on:** Phase 7
**Plans:** 3

Delivered:
- Semantic rule configuration contract.
- SQLite persistence for global and per-user semantic rule configuration.
- Ordered safe executor.
- RTK, Headroom, PII, Rewrite, Caveman, Ponytail, and Filter handlers.
- Startup and hot-reload wiring through runtime semantic rule snapshots.

### Phase 9: Redis Stack + Qdrant Fallback Integration

**Goal:** Integrate Redis Stack for hot cache, atomic rate limiting, Pub/Sub config reload, token-cost aggregation, session/API-key hot state, and Redis VSS fallback when Qdrant is degraded.
**Depends on:** Phase 7
**Plans:** 4

Delivered:
- Redis and local hot-state primitives.
- Atomic limiter and `LimitRule` admission direction.
- SQLite-backed session blacklist and durable accounting reconciliation.
- Redis token-cost aggregation path.
- Redis VSS adapter and Qdrant degradation fallback.
- Typed config event routing.

## Milestone Summary

**Key Decisions:**
- SQLite is the Plan 1 source of truth for relational/security/accounting state.
- Redis is hot-path acceleration and coordination, not the durable source of truth.
- Qdrant remains the primary vector path; Redis VSS is fallback-only.
- Full `LimitRule` unification across all scopes is deferred to a later hardening phase.

**Issues Resolved:**
- LanceDB removed from the default runtime path and isolated for future edge builds.
- Semantic rule execution became deterministic, configurable, and hot-reloadable.
- Redis hot-state verification now uses the real local Redis Stack environment.
- Redis VSS integration was verified against the real local Redis Stack environment after response parsing was fixed.

**Issues Deferred:**
- Full database-wide `LimitRule` unification.
- Advanced routing and observability.
- BFF/Admin Console.
- Multi-node coordination.
- PostgreSQL/pgvector extension.

**Technical Debt:**
- Some Phase 9 tests still rely on local integration services for full confidence.
- Phase 10-13 remain planned and should be re-scoped as the next milestone.
