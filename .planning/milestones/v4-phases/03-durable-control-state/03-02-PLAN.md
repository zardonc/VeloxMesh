<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config/env -> repository | Local development seed input can contain provider connection data and raw API keys. |
| repository -> SQL backend | Durable records and encrypted secrets cross into PostgreSQL or SQLite. |
| app startup -> migration runner | Startup may create schema but must not create production provider records. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Information Disclosure | repository reads | mitigate | Repositories return redacted `ProviderRecord` values and never expose raw secret or ciphertext per D-10 and D-29. |
| T-03-02-02 | Tampering | SQL writes | mitigate | Use parameterized pgx/database-sql statements and optimistic `revision` checks; tests cover stale update conflicts per D-26. |
| T-03-02-03 | Tampering | local-dev seed | mitigate | Seed only when explicitly enabled and durable provider count is zero; tests prove durable records win per D-07 and D-08. |
| T-03-02-04 | Denial of Service | backend unsupported features | mitigate | Capability profiles reject distributed/hot/future features where unsupported instead of silently enabling them per D-06 and D-32. |
| T-03-02-SC | Tampering | Go module installs | mitigate | Blocking human package-legitimacy checkpoint before installing `github.com/jackc/pgx/v5` and `modernc.org/sqlite`. |
</threat_model>
