## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin provider mutation -> Redis pub/sub | Control-plane mutations emit distributed notifications. |
| Redis pub/sub -> other gateway instances | Received messages trigger runtime provider reload. |
| docs/config -> operator behavior | Operators choose Redis or no-Redis consistency mode. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07-01 | Information Disclosure | config-change payloads | mitigate | Payloads include provider ID/action/revision/timestamp only and tests reject secret/ciphertext/prompt/upstream fields per D-35 and D-38. |
| T-03-07-02 | Tampering | pub/sub channel | mitigate | Channel name is namespace-prefixed and configurable per D-36. |
| T-03-07-03 | Denial of Service | reload fanout | mitigate | Subscriber reload uses existing context-bound runtime reload; failures log warning and do not crash data-plane serving per D-38. |
| T-03-07-04 | Repudiation | distributed notification | mitigate | Durable audit from Plan 03-05 remains the source of truth; notifications carry revision/timestamp for correlation per D-21 and D-38. |
| T-03-07-SC | Tampering | package installs | accept | This plan uses the Redis dependency installed and approved in Plan 03-06; no new package installs. |
</threat_model>
