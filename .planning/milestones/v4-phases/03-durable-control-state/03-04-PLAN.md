<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external admin client -> Admin API | Untrusted control-plane requests create and mutate provider configuration. |
| Admin API -> durable repository | Validated mutations and encrypted secrets cross into durable storage. |
| Admin API -> runtime reload | Provider changes can affect live traffic routing immediately. |
| data-plane client -> data-plane API | Existing OpenAI-compatible endpoints must remain separately authenticated. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04-01 | Spoofing | admin routes | mitigate | Dedicated `AdminAuth` compares only `AdminAPIKey`; data-plane `DevAPIKey` is rejected for `/admin/v1` per D-18. |
| T-03-04-02 | Tampering | provider updates | mitigate | Validate required fields, require revision on update, and return `provider_conflict` for stale writes per D-20 and D-26. |
| T-03-04-03 | Information Disclosure | provider responses | mitigate | DTOs expose secret metadata only and tests assert no raw secret/ciphertext/nonce in JSON per D-10 and D-29. |
| T-03-04-04 | Denial of Service | runtime activation | mitigate | Admin service rolls back durable transaction when activation fails, avoiding committed unusable provider config per D-22. |
| T-03-04-05 | Repudiation | provider mutations | mitigate | Plan 03-05 adds audit records for create/update/disable/delete/test; this plan preserves actor label propagation for that integration per D-21. |
| T-03-04-SC | Tampering | package installs | accept | This plan adds no package-manager dependencies. |
</threat_model>
