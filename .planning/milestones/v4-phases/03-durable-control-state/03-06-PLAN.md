<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gateway -> Redis | Hot health and auth-cache metadata leave process memory. |
| data-plane auth -> auth cache | Bearer token validation results may be cached. |
| Redis unavailable -> local fallback | Gateway may degrade to local state when distributed guarantees are unavailable. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06-01 | Information Disclosure | Redis payloads | mitigate | Store no provider secrets, decrypted credentials, encrypted secret blobs, raw bearer tokens, auth headers, raw prompts, or upstream bodies; tests inspect serialized payloads per D-35. |
| T-03-06-02 | Tampering | Redis key namespace | mitigate | Prefix all keys/channels with configured namespace and reject empty namespace per D-36. |
| T-03-06-03 | Denial of Service | Redis outage | mitigate | If configured unavailable, degrade to process-local hot state only when `RedisDegradeToLocal=true`, with clear warning and disabled distributed guarantees per D-34. |
| T-03-06-04 | Replay/Spoofing | auth cache | mitigate | Cache only hashed token identifiers with explicit TTL and validation result; raw bearer tokens are never stored per D-31 and D-37. |
| T-03-06-SC | Tampering | Go module installs | mitigate | Blocking human package-legitimacy checkpoint before installing `github.com/redis/go-redis/v9`. |
</threat_model>
