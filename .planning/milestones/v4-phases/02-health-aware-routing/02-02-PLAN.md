# Phase 2.2 Plan: Go Version Baseline for Official Provider SDKs

<objective>
Confirm and standardize the Go version baseline required by official provider SDKs (especially Anthropic's official SDK), and prove the existing gateway still builds and tests under that baseline without any issues.
</objective>

<context>
Use `.planning/phases/02-health-aware-routing/02-02-CONTEXT.md` as the source of phase decisions.

Important locked decisions:
- Move native Anthropic/Gemini adapter implementation to Phase 2.3.
- Phase 2.2 only handles Go version/toolchain readiness and verification.
- Anthropic's official Go SDK requires Go 1.24+.
- Current `go.mod` already declares `go 1.26.1`, which satisfies this requirement.
</context>

<scope>
## Must Build
- Verify the selected Go baseline (1.26.1) supports Anthropic official SDK (needs 1.24+).
- Update stale Go version references in docs, CI, Makefile, Dockerfile, or scripts if found.
- Run `go mod tidy`.
- Run `go test ./...`.

## Must Not Build
- Native Anthropic adapter using official SDK.
- Native Gemini adapter.
- Provider response normalization.
</scope>

<tasks>

## 1. Verify Go Baseline and Stale References

type: investigation

action:
- Run `go version` to confirm the local environment meets the Go 1.26.1 requirement.
- Search the codebase for any remaining "1.22" references (e.g. in `Makefile`, `.github/workflows`, `Dockerfile`) and update them to "1.26.1". Note: `README.md` has already been updated.

verify:
- Local environment uses the correct Go version.
- No stale 1.22 references exist.

acceptance_criteria:
- The project is fully standardized on the Go 1.26.1 baseline.

## 2. Execute Toolchain and Tests

type: testing

action:
- Run `go mod tidy` to ensure `go.mod` and `go.sum` are clean and up-to-date.
- Run `go fmt ./...` to ensure code formatting consistency.
- Run `go test ./...` to verify all existing unit and integration tests pass successfully under the Go 1.26.1 toolchain.

verify:
- `go mod tidy` exits cleanly.
- `go test ./...` passes without errors.

acceptance_criteria:
- Existing codebase is proven to be stable and ready for Phase 2.3 under the required Go baseline.

</tasks>

<verification>
Run these checks before marking Phase 2.2 complete:
1. `go version` confirms Go 1.26.1+ is active.
2. `go test ./...` output shows all tests passing.
3. No `.env`, `Makefile`, or `README.md` files still mention Go 1.22.
</verification>

<success_criteria>
- Go baseline is documented and compatible with Anthropic official SDK requirements (1.24+).
- Existing code builds and tests under the selected Go version.
- Stale Go 1.22 references are removed or updated where relevant.
- Phase 2.3 can proceed with Anthropic official SDK adoption without reopening Go version questions.
- No provider adapter implementation is added in Phase 2.2.
</success_criteria>
