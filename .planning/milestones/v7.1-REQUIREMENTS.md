## v7.1 Requirements

### Routing

- [x] **ROUT-01**: Gateway can route provider requests through a Composite Score Router. (Verified via test suite)
- [x] **ROUT-02**: Gateway can score routing candidates using latency, pending requests, error rates, cost, and health bonus signals. (Verified via test suite)
- [x] **ROUT-03**: Gateway can normalize routing signals with z-score normalization before weighting. (Verified via test suite)

### Observability

- [x] **OBS-01**: Operators can inspect OpenTelemetry traces for TTFT, TPOT, end-to-end latency, and cache-hit behavior. (Verified via test suite and tracing setup)
- [x] **OBS-02**: Operators can scrape Prometheus histogram metrics for routing and request timing. (Verified via test suite and metrics route)
