<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Scheduler -> Predictor worker | Safe `TaskFeature` scalars cross into local model runtime. |
| Runtime artifact -> Scheduler | Manifest controls tensor mapping and must fail fast on schema drift. |
| Predictor signals -> Scheduler policy | Model-native signals influence confidence and uncertainty but not priority or queue ownership. |
| Worker process -> host | Python child process must be local, bounded, restartable, and observable. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-04-01 | Tampering | manifest.json | mitigate | Validate protocol, quantiles, feature schema, training hash presence, and compatible scheduler version before prediction. |
| T-18-04-02 | Denial of Service | Python worker | mitigate | Long-lived worker, call timeout, breaker, restart backoff, health probes, and NoopPredictor fallback. |
| T-18-04-03 | Information Disclosure | predictor RPC/logs/metrics | mitigate | Send only safe `TaskFeature` fields; keep raw prompts, embeddings, auth headers, API keys, provider secrets, tenant IDs, and payloads out. |
| T-18-04-04 | Spoofing | predictor signals | mitigate | Scheduler treats signals as model evidence only; trusted priority and queue ownership remain Gateway/Scheduler policy. |
| T-18-04-05 | Repudiation | quality evidence | mitigate | Record predictor version, anomaly status, fallback reason, and per-task predictor error class with low-cardinality labels. |
</threat_model>
