<threat_model>
STRIDE register:
- Spoofing: rollout controls must be admin-only. Disposition: handlers are registered behind existing admin authentication middleware and tests reject unauthenticated requests.
- Tampering: rollout percent changes must be validated and auditable. Disposition: PATCH accepts only 0..100 and records sanitized old/new percent audit metadata.
- Information disclosure: rollout status and alerts must not expose tenant/API-key identity, raw prompts, raw payloads, task IDs, provider secrets, or model internals. Disposition: status uses aggregate rollups only and handler tests assert forbidden fields are absent.
- Denial of service: automatic rollback could oscillate traffic or mask diagnostics. Disposition: alerting never changes rollout state; operators manually set percent to 0.
- Repudiation: rollout changes need operator-visible history. Disposition: sanitized admin audit events include old_percent, new_percent, outcome, and timestamp.
</threat_model>
