<threat_model>
STRIDE register:
- Information disclosure: rollout assignment and backend metadata must not leak tenant/API-key identity, raw prompts, raw payloads, scheduler topology, or model internals to public data-plane clients. Disposition: backend metadata stays internal and data-plane grep checks prove no new response fields or headers.
- Denial of service: ONNX failure must not block task execution. Disposition: ONNX-selected failures fall back to heuristic, then the existing FIFO path.
- Tampering: rollout percentage must stay bounded to valid values. Disposition: config validation rejects values outside 0..100 and tests cover boundary values.
- Spoofing: assignment must not be controlled by tenant/API-key identity. Disposition: deterministic task-level hash uses `TaskID` and non-identity fallback inputs only.
</threat_model>
