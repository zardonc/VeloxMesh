<threat_model>
STRIDE register:
- Information disclosure: quality evidence must not contain tenant/API-key identity, raw prompts, raw payloads, auth headers, API keys, provider secrets, or high-cardinality labels. Disposition: schema allowlist, metric label tests, and migration grep checks.
- Denial of service: quality persistence must not block or fail public responses. Disposition: recorder writes are best-effort by default and tested for response isolation.
- Tampering: quality rollups must be computed from gateway-owned completion labels, not public client input. Disposition: recorder is wired after internal task completion and no public write API is added.
- Repudiation: rollups need enough safe evidence for audit without sensitive data. Disposition: bucket timestamps, scheduler type/version/task type, aggregate counts, and safe sample IDs are retained.
</threat_model>
