<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin client -> test-connection action | External request can cause a provider validation call. |
| Admin mutation -> idempotency store | Request fingerprints and responses are persisted for replay. |
| Admin mutation -> audit store | Action metadata is persisted for accountability. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05-01 | Information Disclosure | test-connection response | mitigate | Return stable generic messages and never include raw upstream bodies, prompts, auth headers, API keys, ciphertext, or nonce per D-17, D-24, and D-29. |
| T-03-05-02 | Tampering | idempotency replay | mitigate | Fingerprint method/path/canonical body with secret fields redacted; mismatched replay returns 409 per D-28. |
| T-03-05-03 | Repudiation | provider mutations | mitigate | Record actor label, timestamp, target provider ID, action, outcome, and safe metadata for every provider create/update/disable/delete/test per D-21. |
| T-03-05-04 | Denial of Service | audit growth | mitigate | Configurable retention purge removes old audit records only, with conservative local/dev defaults per D-27. |
| T-03-05-SC | Tampering | package installs | accept | This plan adds no package-manager dependencies. |
</threat_model>
