<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Authorization header -> auth cache | Bearer token is hashed before cache interaction. |
| session hash -> Redis/SQLite blacklist | Revocation state affects request/session acceptance. |
| provider usage -> Redis aggregation | Billing-related counters cross into hot state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-03-01 | Information Disclosure | auth cache | mitigate | Cache only token hash-derived keys and safe identity envelope; never raw keys/headers. |
| T-09-03-02 | Repudiation | cost aggregation | mitigate | Keep SQLite settlement as authoritative audit record; Redis aggregation is derivative. |
| T-09-03-03 | Tampering | session blacklist | mitigate | Store revocations in SQLite and mirror to Redis with expiry. |
| T-09-03-04 | Scope creep | BFF sessions | mitigate | Provide blacklist primitive only; defer full JWT/session flow to Phase 11. |
</threat_model>
