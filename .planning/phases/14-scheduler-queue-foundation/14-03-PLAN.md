- Information disclosure: Feature extraction and Scheduler metrics must not expose raw prompts, messages, tool args, auth headers, API keys, provider secrets, task payloads, or high-cardinality IDs. Disposition: bounded enum/numeric outputs and low-cardinality metric labels only.
- Tampering/elevation of privilege: Prompt content must not raise priority. Disposition: feature extraction accepts resolved priority as an input and tests prove prompt text cannot elevate it.
- Denial of service: Scheduler scoring must stay bounded and stateless. Disposition: table/rule scoring only, no Qdrant, ONNX, Redis, SQLite, or network calls inside BatchScoreTasks.
</threat_model>
