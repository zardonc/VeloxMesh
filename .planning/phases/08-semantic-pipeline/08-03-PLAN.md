<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated client -> gateway | User chat content enters semantic processing. |
| admin API -> rule store | Admin-provided config changes persisted behavior for users. |
| gateway -> semantic cache | Processed request/response content may become cache keys or cache values. |
| hot-state publish -> subscribers | Config change notifications cross process boundaries. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-03-01 | Spoofing | admin semantic endpoints | mitigate | Reuse existing `middleware.AdminAuth` on every semantic config route. |
| T-08-03-02 | Tampering | admin writes | mitigate | Validate writes through the Plan 08-01 config validator before saving. |
| T-08-03-03 | Information Disclosure | logs and hot-state messages | mitigate | Publish only action metadata and log no prompts, secrets, raw rule payloads, API keys, or authorization headers. |
| T-08-03-04 | Denial of Service | streaming response transforms | mitigate | Preserve pass-through streaming when response-side rules are disabled; when response-side rules require buffering, enforce `max_stream_transform_bytes` default 1 MiB and pass through with a sanitized skip log if exceeded. |
| T-08-03-SC | Tampering | package dependencies | accept | No new package-manager install in this plan. |
</threat_model>
