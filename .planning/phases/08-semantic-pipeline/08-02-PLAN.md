<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client request -> pipeline | Untrusted chat content enters rule handlers. |
| provider response -> pipeline | Upstream model output is transformed or filtered before returning to the client. |
| pipeline -> logs | Handler failures produce operational logs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-02-01 | Information Disclosure | PII and exception logging | mitigate | Redact before Rewrite/RTK and log only rule/user/request/error metadata without raw prompts or sensitive values. |
| T-08-02-02 | Denial of Service | handler execution | mitigate | Handler errors skip only that handler and continue per D-13; filter blocks are intentional decisions. |
| T-08-02-03 | Tampering | style handlers | mitigate | Enforce Caveman/Ponytail mutual exclusion and request-rewrite opt-in through the resolved config. |
| T-08-02-SC | Tampering | package dependencies | accept | No new package-manager install; use Go stdlib regex/string behavior and existing project packages. |
</threat_model>
