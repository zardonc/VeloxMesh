## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Redis coordination -> executor | Redis lock state controls whether a popped task may execute. |
| Queue state -> metrics | Queue admission and depth are exported to Prometheus. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-02-01 | Tampering | Redis execution lock | mitigate | Use `SET NX` before execution, release after delivery, and skip execution when another worker owns the task. |
| T-20-02-02 | Information Disclosure | queue metrics | mitigate | Emit only backend, priority, outcome, reason, and depth; never emit task text or provider payloads. |
| T-20-02-03 | Denial of Service | executor slots | mitigate | Clamp configured concurrency to at least one and preserve context cancellation behavior. |
</threat_model>
