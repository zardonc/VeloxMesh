# Phase 7 Plan 01: SQLite-First Foundation and Adapter Seams

## Objective
- **What:** Make the gateway's Plan 1 foundation explicit and runnable: SQLite-first durable state, local/no-op adapters, fallback log schema, optional LanceDB semantic cache, and clean seams for later Redis and PostgreSQL extensions.
- **Why:** Phase 8-12 depend on a stable storage/runtime baseline. Without this, every later feature risks reintroducing PostgreSQL-first or Redis-required assumptions.
- **Output:** Narrow adapter contracts, SQLite startup/default hardening, fallback log migration, docs, and focused tests.
