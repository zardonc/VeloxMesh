# Phase 7 Plan 01: SQLite-First Foundation and Adapter Seams

## Objective
- **What:** Make the gateway's v2.1 Plan 1 foundation explicit and runnable: SQLite authoritative relational state, Redis-compatible hot-state seams, Qdrant primary vector/semantic-cache path, degraded/noop vector behavior, fallback log schema, and clean seams for later LanceDB edge and PostgreSQL extensions.
- **Why:** LanceDB blocked current development and conflicts with the new cross-platform mainline. Phase 8-12 depend on a stable Qdrant-centered storage/runtime baseline.
- **Output:** Narrow adapter contracts, SQLite startup/default hardening, Qdrant vector/semantic-cache planning path, LanceDB cancellation/defer notes, fallback log migration, docs, and focused tests.
