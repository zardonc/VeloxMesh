## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Queue backend -> promotion logic | Redis or memory queue exposes bounded task IDs and scores for pre-pop inspection. |
| Auth context/request -> task snapshot | Only trusted identity and safe structured feature fields are stored for promotion. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-02-01 | Information Disclosure | ResultRegistry task snapshots | mitigate | Store tenant ID/class and safe TaskFeature fields only; exclude prompts, messages, API keys, auth headers, provider payloads, embeddings, semantic-cache payloads, and raw task text. |
| T-19-02-02 | Tampering | SLAPromoter score update | mitigate | Reuse QueueBackend.Push replacement, compute a same-priority score with math.Nextafter, and block when a promotion would cross higher-priority boundaries. |
| T-19-02-03 | Denial of Service | Queue.PeekMin | mitigate | Inspect only CandidateWindow items and treat limit less than 1 as empty. |
| T-19-02-04 | Elevation of Privilege | Priority handling | mitigate | Match only trusted TenantID/TenantClass plus model_class/request_kind; never read prompt-derived urgency fields and never rerun quota to borrow high priority. |
</threat_model>
